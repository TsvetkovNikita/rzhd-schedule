
# agents.md

## Архитектура проекта
- `app.py` — entry point Flask-приложения.
- `core/` — конфиг и доменные модели.
- `clients/` — внешние HTTP-клиенты, включая Яндекс.Расписания.
- `db/` — SQLAlchemy engine, ORM и репозитории.
- `services/` — бизнес-логика чтения, импорта и кэша.
- `tasks/` — фоновые задачи и расписание импорта.
- `web/` — auth-декораторы и web-специфичная логика.
- `templates/`, `static/` — HTML, JS и CSS skins.
- `alembic/` — миграции БД. Создание схемы через Alembic, а не вручную.

## Правила слоёв
1. Маршруты и background-задачи не обращаются к БД напрямую.
2. Доступ к БД идёт только через `repository_gateway.session_scope()`.
3. Репозитории не вызывают HTTP-клиенты.
4. `services/` координируют внешние клиенты, репозитории и доменные модели.
5. UI и API читают расписание только из локальной БД.

## Импорт расписания
1. При старте приложения выполняется первый импорт из Яндекс.Расписаний, если `scheduler.run_on_startup=true`.
2. Затем scheduler повторяет импорт с интервалом `scheduler.interval_seconds`.
3. Каждый импорт сохраняется как отдельный batch в `schedule_import_batches`.
4. Строки batch сохраняются в `schedule_import_rows`.
5. Табло всегда показывает последний успешный batch.

## Стили и skin-файлы
- Базовые общие правила лежат в `static/css/base.css`.
- Переключаемые стили лежат в `static/css/skins/`.
- Доступные skins: `classic`, `full_hd`, `half_hd`, `contrast`.
- `half_hd` считается отдельным поддерживаемым режимом и не должен ломаться при правках.
- Новые адаптации добавлять отдельным skin-файлом, а не раздувать один CSS.

## Миграции
- Любое изменение ORM-моделей требует отдельной Alembic migration.
- Baseline migration: `0001_baseline`.
- Revision ID держать коротким, не длиннее 32 символов.
- В `alembic/env.py` URL перед передачей в Alembic экранируется через `.replace("%", "%%")`.

## Что не делать
- Не возвращать UI к прямому чтению из Яндекса.
- Не вызывать `Base.metadata.create_all()` как основной способ создания схемы.
- Не смешивать CSS разных экранных режимов в одном skin без необходимости.
